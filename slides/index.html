01 the shift everyone's standing on

Stop prompting agents.
Start designing the loop they run inside.

June 2026 gave it a name — loop engineering (Osmani, Cherny). The skill is no longer the perfect prompt; it's the generate → check → revise loop with a real exit condition. As one post that hit 6.5M views put it, eleven days before this deadline:

"You shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents."

02 the gap nobody's closed

Every coding tool ships one loop —
and runs it for everything.

Planner → Coder → Reviewer. The identical pipeline.

A one-line bugfix and an auth change that can leak sessions get the same gates, the same reviewers, the same stopping rule. That's vibe coding with extra steps — and building the loop by hand is still expert work nobody's automated.

03 what ouroboros is

A band of agents that engineers the loop a task needs — then runs it.

Point it at a task. The room designs the verification loop for that task — which checks gate it, which critics vote, whether a human must sign — then runs it: generate a revision, let the critics attack, run the real checks, revise until a real exit condition holds.

The snake that eats its own tail: the oldest symbol for a loop that feeds itself until the code is whole. The thing it produces is an artifact you can read — the LoopSpec — not just a diff you have to trust.

05 why band is the coordination layer — not a wrapper

Take Band out and this is impossible.

① Dynamic recruitment

The room decides at runtime it needs a voice nobody added up front, and adds it — band_add_participant. The membership of the room grows to fit the task. This is the most Band-native moment in the system.

② Design + execution, one trail

The loop's design and its execution live on the same audit trail — every handoff is an @mention. You can read the loop a room built before you trust the code it produced. The transcript is the record.

  band room · handoff chain

06 not a mock — the real loop, bouncing

The checks really run. Red is real. Green is earned.

  transcript · "Fix off-by-one in pagination offset" · real subprocess QA

QA executes generated code in a sandboxed subprocess. Revision 0 fails on a genuine AssertionError; the room bounces it back; revision 1 goes green and ships. No fixture fakes the result.

07 the real bug, the real fix, the real failure

revision 0 — buggydef page(items, page_num, size): start = page_num * size end = start + size - 1 # drops last row return items[start:end]
revision 1 — shippeddef page(items, page_num, size): start = page_num * size end = start + size # slice end is exclusive return items[start:end]
repro-test ❌ AssertionError: page returned a short page assert len(page(list(range(10)), 0, 3)) == 3

If the loop can't make your tests pass in budget, it stops and hands you a verified failure — not a confident wrong answer. That's the failure mode of agentic coding, closed.
